# Learning Checklist

- [ ] I can pass a request options object as the second argument to `fetch`.
- [ ] I can select the `POST` method.
- [ ] I can serialize a JavaScript object with `JSON.stringify`.
- [ ] I can place the serialized JSON in the request body.
- [ ] I can set the request's `Content-Type` header to `application/json`.
- [ ] I can distinguish request headers from response headers.
- [ ] I can read a response header with `response.headers.get`.
- [ ] I can inspect `response.status` and check `response.ok`.
- [ ] I understand why successful creation may return `201` instead of `200`.
- [ ] I understand that this practice API simulates rather than persists writes.

You have met the lesson goals when you can describe every part of the outgoing
request separately from every part of the incoming response.
