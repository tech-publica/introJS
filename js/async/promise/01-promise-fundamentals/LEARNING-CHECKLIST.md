# Learning Checklist

- [ ] I can explain that a promise is available now but represents a future outcome.
- [ ] I can distinguish a promise object from its eventual fulfillment value.
- [ ] I can name and describe the `pending`, `fulfilled`, and `rejected` states.
- [ ] I understand that a promise settles only once.
- [ ] I can explain when the promise executor runs.
- [ ] I can describe the roles of `resolve` and `reject`.
- [ ] I can use `then`, `catch`, and `finally` for their intended outcomes.
- [ ] I understand that registering a handler does not pause JavaScript.
- [ ] I can predict why promise handlers run after current synchronous code.

Do not move on until you can explain the difference between “the promise was
returned” and “the promise was fulfilled.”
